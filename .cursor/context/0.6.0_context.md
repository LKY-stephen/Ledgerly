# Context Snapshot

## Version

- version: 0.6.0
- updated_at: 2026-04-22
- scope: ship the local-first home assistant with configurable LLM chat, live tool-call updates, and natural-language quick-entry for ledger records

## Active Decisions

- Monorepo remains the baseline.
- Frontend stays on Expo + React Native + Expo Router.
- There is still no backend in the current phase.
- Assistant writes must continue to resolve against the existing local SQLite schema; no remote sync or backend tables were added.
- Quick-entry assistant flows may use safe default `source` / `target` labels when the user omits payer/payee detail, rather than blocking lightweight bookkeeping.
- AI provider selection stays device-local; OpenAI is the default assistant provider for fresh installs, with Gemini and Infer remaining configurable.

## Implemented Structure

- `packages/agent/src/agent-session.ts`: live message subscriptions, context refresh after tool writes, and prompt guidance for quick-entry defaults.
- `packages/agent/src/tools.ts`: assistant-side default `source` / `target` fallbacks for create-record calls.
- `apps/mobile/src/features/agent/*`: wired the home assistant session to local storage and app-shell configuration.
- `apps/mobile/src/features/home/home-screen.tsx`: home tab now exposes an expandable assistant panel, with setup guidance when AI config is missing.
- `apps/mobile/src/features/profile/profile-screen.tsx`: AI provider and API key settings are visible again so the assistant can be activated from product UI.
- `tests/smoke/README.md`: manual smoke now includes the home assistant ask/add-record loop.
