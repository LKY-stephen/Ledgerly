# Context Snapshot

## Version

- version: 0.7.0
- updated_at: 2026-04-23
- scope: document the supported/current AI-provider architecture as env-backed middleware with separate assistant and parse/planner consumers

## Active Decisions

- Monorepo remains the baseline.
- Frontend stays on Expo + React Native + Expo Router.
- There is still no backend in the current phase.
- Structured records stay in SQLite and documents stay in the local file vault.
- `ai_provider` is documented as an env-backed middleware concept rather than as a persisted runtime selector.
- Assistant chat and upload parse/planner are separate consumers of that middleware and do not share identical Infer completeness rules.
- Public/current docs describe environment-based configuration only.
- Google OAuth remains hidden/internal and is not part of the current supported public AI-provider contract.
- Existing settings-driven and device-state AI paths remain compatibility/runtime residue until a later PRD decides whether the code should fully converge to the published contract.

## Implemented Structure

- `docs/architecture.md`: documents the env-backed middleware, the assistant vs parse/planner split, and the transport / precedence differences.
- `docs/contracts/local-storage.md`: frames persisted AI-related device state as compatibility/internal state instead of the supported selector contract.
- `docs/upload-planner-workflow.md`: documents parse/planner as its own middleware consumer with OpenAI -> Infer -> Gemini precedence and Infer base-URL completeness requirements.
- `docs/testing.md`: adds smoke expectations for the assistant vs parse/planner Infer split.
- `tests/smoke/README.md`: moves the supported smoke path to environment-based provider configuration and removes Google OAuth from the current public verification contract.
- `packages/storage/src/contracts.ts`: wording-only contract metadata aligns the device-state summaries with the published/current middleware story.
